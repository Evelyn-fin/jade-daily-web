# 飞书快速部署清单

## ✅ 部署前检查

- [ ] 确保后端服务正在运行：`cd server && pnpm run dev`
- [ ] 确认后端端口 9091 可访问
- [ ] 已注册 Vercel/Netlify 账号

## 🚀 5分钟快速部署

### 1️⃣ 构建前端（1分钟）

```bash
cd /workspace/projects
node build-web.js
```

### 2️⃣ 部署到 Vercel（3分钟）

1. 访问 https://vercel.com 并登录
2. 点击 "New Project"
3. 推送 `dist-web` 到 GitHub 或直接上传文件夹
4. 配置环境变量：
   - Name: `EXPO_PUBLIC_BACKEND_BASE_URL`
   - Value: `http://localhost:9091`（临时）或你的内网穿透地址
5. 点击 "Deploy"
6. 等待部署完成，获得 URL

### 3️⃣ 配置内网穿透（5分钟，可选）

如果需要在飞书中使用，必须配置内网穿透：

**使用 ngrok（测试）：**
```bash
# 安装
brew install ngrok  # macOS
# 或从官网下载 Windows 版本

# 启动
ngrok http 9091

# 得到临时URL：https://xxx.ngrok.io
```

**更新 Vercel 环境变量：**
- 将 `EXPO_PUBLIC_BACKEND_BASE_URL` 改为 ngrok URL
- 例如：`https://abc123.ngrok.io/api/v1`

### 4️⃣ 在飞书中添加应用（2分钟）

1. 访问 https://open.feishu.cn/app
2. 创建 "网页应用"
3. 填写信息：
   - 名称：Jade's Daily Planner
   - URL：你的 Vercel URL
4. 发布应用

## 🎯 验证部署

- [ ] 在浏览器打开 Vercel URL，页面正常显示
- [ ] 可以创建待办事项
- [ ] 可以创建灵感并AI生成待办
- [ ] 在飞书中可以正常打开
- [ ] 所有功能正常工作

## 🐛 快速修复

### 前端正常但API失败

检查：
1. 后端是否运行：`curl http://localhost:9091/api/v1/health`
2. 环境变量是否正确：在 Vercel 控制台查看
3. CORS是否配置：检查后端 `src/index.ts`

### 飞书中显示空白

检查：
1. URL是否使用HTTPS
2. 浏览器控制台是否有错误（F12）
3. 尝试在普通浏览器访问同一URL

## 🎉 最简单的部署方式（无需内网穿透）

如果你只是想快速测试：

1. **部署前端到 Vercel**（如上）
2. **环境变量使用本地地址**：
   ```
   EXPO_PUBLIC_BACKEND_BASE_URL = http://localhost:9091
   ```
3. **在同一台电脑上打开飞书客户端**
4. **在飞书中使用应用**

这样前端在云端，后端在本地，但飞书客户端可以访问本地API（仅限同一台电脑）。
