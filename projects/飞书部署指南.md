# 飞书部署指南 - Jade's Daily Planner

本指南将帮助你将 Jade's Daily Planner 部署到飞书，让团队可以在飞书中使用这个应用。

## 🚀 部署步骤

### 第一步：构建前端Web版本

在项目根目录执行：

```bash
# 方法1：使用构建脚本
node build-web.js

# 方法2：手动构建
cd client
npx expo export --platform web --output-dir ../dist-web
```

构建完成后，会在项目根目录生成 `dist-web` 文件夹。

### 第二步：部署到云服务

选择以下任一方式部署前端：

#### 方式A：Vercel（推荐，最简单）

1. 访问 [Vercel](https://vercel.com) 并注册账号
2. 点击 "New Project"
3. 将 `dist-web` 文件夹推送到 GitHub/GitLab
4. 在 Vercel 中导入仓库
5. 配置构建设置：
   - Framework Preset: Other
   - Build Command: 留空（已预构建）
   - Output Directory: `./`
   - Root Directory: 留空

6. 部署完成后，Vercel 会提供一个 URL，例如：`https://jade-daily-planner.vercel.app`

#### 方式B：Netlify

1. 访问 [Netlify](https://netlify.com) 并注册账号
2. 拖拽 `dist-web` 文件夹到 Netlify 控制台
3. 部署完成，获得 URL

#### 方式C：腾讯云 COS

1. 访问 [腾讯云 COS](https://cloud.tencent.com/product/cos)
2. 创建存储桶，设置为静态网站托管
3. 上传 `dist-web` 文件夹内容
4. 获取静态网站域名

#### 方式D：阿里云 OSS

1. 访问 [阿里云 OSS](https://www.aliyun.com/product/oss)
2. 创建 Bucket，设置为静态网站
3. 上传 `dist-web` 文件夹内容
4. 获取域名

### 第三步：配置后端访问

由于后端运行在本地，需要配置内网穿透让前端可以访问：

#### 方式1：使用 frp（推荐）

**服务器端（你的本地电脑）：**

1. 下载 frp：https://github.com/fatedier/frp/releases
2. 配置 `frpc.ini`：

```ini
[common]
server_addr = your-frp-server.com  # 你的frp服务器
server_port = 7000

[jade-api]
type = tcp
local_ip = 127.0.0.1
local_port = 9091
remote_port = 6000  # 公网端口
```

3. 启动 frp 客户端：
```bash
./frpc -c frpc.ini
```

**你的后端API地址变为：**
```
http://your-frp-server.com:6000/api/v1
```

#### 方式2：使用 ngrok（测试用）

```bash
# 安装 ngrok
# macOS: brew install ngrok
# Windows: 下载安装包

# 启动内网穿透
ngrok http 9091
```

会得到一个临时URL，例如：
```
https://abc123.ngrok.io
```

**注意：ngrok 的URL是临时的，重启后会变化。**

#### 方式3：部署后端到云服务器（生产环境）

1. 购买云服务器（腾讯云/阿里云等）
2. 上传代码并启动后端服务
3. 配置防火墙开放 9091 端口
4. 获得公网IP或域名

### 第四步：更新前端API地址

前端代码中使用了环境变量 `EXPO_PUBLIC_BACKEND_BASE_URL`，需要在云服务中配置：

#### Vercel 配置：

在项目根目录创建 `vercel.json`：

```json
{
  "build": {
    "env": {
      "EXPO_PUBLIC_BACKEND_BASE_URL": "http://your-frp-server.com:6000"
    }
  }
}
```

或者直接在 Vercel 控制台的 Settings > Environment Variables 中添加。

#### Netlify 配置：

在 Netlify 控制台的 Site settings > Environment variables 中添加：
```
EXPO_PUBLIC_BACKEND_BASE_URL = http://your-frp-server.com:6000
```

### 第五步：在飞书中创建网页应用

1. 打开飞书工作台：https://open.feishu.cn/app
2. 点击"创建应用" → "网页应用"
3. 填写基本信息：
   - 应用名称：Jade's Daily Planner
   - 应用描述：日常管理助手 - 每日语录 + 待办清单 + AI智能规划
   - 应用图标：上传你的应用图标
4. 配置网页地址：
   - PC端：填写前端部署的URL（如：`https://jade-daily-planner.vercel.app`）
   - 移动端：填写相同的URL
5. 配置权限：
   - 无需特殊权限，直接发布即可
6. 发布应用：
   - 提交审核
   - 审核通过后，可在飞书工作台中使用

## 🔧 常见问题

### Q1: 前端可以访问，但API请求失败

**原因：** 后端API地址配置错误或后端服务未启动

**解决：**
1. 检查 `EXPO_PUBLIC_BACKEND_BASE_URL` 是否正确
2. 确保后端服务正在运行：`cd server && pnpm run dev`
3. 检查内网穿透是否正常
4. 测试API是否可访问：`curl http://your-api-url/api/v1/health`

### Q2: 飞书中打开显示空白

**原因：** 飞书可能阻止了某些资源或脚本

**解决：**
1. 检查浏览器控制台是否有错误
2. 确保前端URL使用 HTTPS
3. 检查CORS配置是否正确
4. 尝试在普通浏览器中访问同一URL

### Q3: AI生成功能不工作

**原因：** API调用失败或超时

**解决：**
1. 检查后端日志，查看AI服务调用情况
2. 确认豆包AI API密钥配置正确
3. 增加超时时间（后端配置）
4. 检查网络连接

### Q4: ngrok URL经常变化

**原因：** ngrok 免费版URL是临时的

**解决：**
- 升级 ngrok 付费版，固定域名
- 使用 frp 配置固定端口
- 部署后端到云服务器

## 🚀 生产环境建议

### 1. 使用固定域名

购买域名并配置：
- 前端：`app.yourdomain.com`
- 后端：`api.yourdomain.com`

### 2. 启用HTTPS

- 前端：云服务自动提供HTTPS
- 后端：使用 Nginx 反向代理 + Let's Encrypt 证书

### 3. 数据库备份

定期备份 SQLite 数据库：
```bash
# 备份
cp server/data/jade-daily.db backup/jade-daily-$(date +%Y%m%d).db

# 恢复
cp backup/jade-daily-20260318.db server/data/jade-daily.db
```

### 4. 监控和日志

- 使用日志收集服务（如 Sentry）
- 配置健康检查和自动重启
- 监控API响应时间
